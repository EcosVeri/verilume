# Code of Conduct

## Our Pledge

We want Verilume to be a welcoming, respectful, and productive open-source community.

Everyone participating in this project is expected to help create an environment that is inclusive, transparent, and safe for contributors of all backgrounds and experience levels.

## Expected Behavior

Examples of behavior that helps the community include:

- Being respectful and constructive in discussions.
- Welcoming newcomers and helping them get oriented.
- Giving feedback that focuses on ideas, code, documentation, and project goals.
- Accepting responsibility for mistakes and correcting them.
- Respecting different viewpoints, experiences, and levels of expertise.
- Prioritizing the health of the project and its users.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language, imagery, or attention.
- Trolling, insults, or deliberately disruptive conduct.
- Publishing private information without explicit permission.
- Repeatedly ignoring maintainer guidance.
- Any conduct that would reasonably make someone feel unsafe or unwelcome.

## Scope

This Code of Conduct applies in all Verilume project spaces, including:

- GitHub issues and pull requests
- Discussions
- Documentation
- Community chat spaces
- Project meetings
- Public communication when representing the project

## Reporting

If you experience or witness unacceptable behavior, please report it to the project maintainers.

When reporting, include as much relevant context as possible:

- What happened
- Where it happened
- Who was involved
- Any links, screenshots, or logs that help explain the situation

Reports will be handled with care and confidentiality.

## Enforcement

Maintainers may take action when this Code of Conduct is violated. Actions may include:

- A private warning
- A request to edit or remove content
- Temporary restriction from project spaces
- Permanent removal from project spaces
- Reporting to platform administrators when necessary

Enforcement decisions will aim to protect contributors, preserve project health, and encourage accountability.

## Maintainer Responsibilities

Project maintainers are responsible for:

- Clarifying acceptable behavior
- Responding to reports
- Taking proportionate action when needed
- Applying this Code of Conduct fairly
- Protecting the privacy and safety of affected contributors

## Our Commitment

Verilume is built around trust, transparency, and evidence. The same values should guide how we collaborate.

We welcome contributions from people of all backgrounds and ask everyone to help keep the community kind, focused, and open.
