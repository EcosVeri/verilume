# 🔭 Verilume Vision

> **A local-first desktop knowledge assistant with transparent sources and open model choice.**

---

# Our Vision

Artificial intelligence is transforming how people access and interact with information. Yet many AI assistants remain closed, opaque, dependent on cloud services, and unable to explain where their answers come from.

**Verilume exists to change that.**

Our vision is to build an open, transparent, privacy-first AI knowledge platform
that combines local documents, trusted web information, local models, and
independently developed online models into a single intelligent assistant.

Verilume should be easy enough for a non-programmer to install, launch, and use:

* a single downloadable macOS desktop app;
* a Python package installable from PyPI;
* a GitHub install path for developers and early adopters;
* a Windows executable or installer if Windows packaging proves reliable.

Every answer should be:

* **Grounded** in evidence.
* **Transparent** about its sources.
* **Current** when information changes.
* **Private** whenever possible.
* **Independent** from any one model vendor or hosted-only API.
* **Open** for everyone to inspect, improve, and extend.

# Our Principles

## 📖 Transparency

Every answer should explain where it came from.

Whenever possible Verilume identifies whether information originated from

* local documents
* AI model knowledge
* trusted web sources
* government publications
* research papers
* news sources

Users should never wonder:

> "How do you know that?"

---

## 🔒 Privacy First

Private documents should stay private.

Whenever possible:

* inference should run locally
* embeddings should remain local
* vector databases should remain local

Cloud services should always be optional—not mandatory. Verilume should support
local backends like Ollama and independently developed online providers alongside
larger hosted model services.

---

## 🧩 Open Model Choice

Users should be able to choose the model path that fits their situation.

Verilume should support

* Hugging Face models;
* Ollama local models;
* independently developed online model providers;
* future local runtimes and open inference servers.

The product should avoid hard dependency on hosted-only APIs. Independently
developed online models can be useful, but they should be one option, not the
foundation of the system.

---

## 🌍 Hybrid Intelligence

No single source is sufficient.

Verilume intelligently combines

* local knowledge
* AI model knowledge
* trusted web retrieval

to produce the most reliable answer available.

For rapidly changing information, Verilume prioritizes trusted web sources.

For private knowledge, Verilume prioritizes local documents.

---

## 🧠 Multimodal Understanding

Knowledge is more than text.

Future versions of Verilume will understand

* scanned PDFs
* images
* diagrams
* handwritten notes
* tables
* graphs
* equations
* presentations
* screenshots
* forms

Verilume should understand information the same way humans do.

---

## 📚 Research Friendly

Researchers deserve better tools.

Verilume aims to become an AI assistant capable of

* reading papers
* summarizing literature
* finding contradictions
* extracting references
* generating reports
* explaining mathematical concepts
* assisting scientific discovery

without sacrificing transparency.

---

Organizations need trustworthy AI.

Verilume is designed to support

through explainable retrieval rather than black-box responses.
